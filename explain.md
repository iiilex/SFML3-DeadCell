# 我需要什么呢?

## 一套物理系统

包括了在这个游戏世界中的各种物理模型

0. 世界中的物理参数
   1. 重力加速度
   2. 摩擦系数

1. 一套集成的物理参数 PhysicsData
   1. 世界中的坐标 (左上为(0,0), 向右为x,向下为y)
   2. 速度(x方向, y方向)
   3. 碰撞箱
   4. 是否撞墙
   5. 是否落地
   
2. 一套物理世界的判断更新规则 PhysicsSystem
   1. 根据重力加速度更新y方向速度
   2. 按照速度更新当前坐标
   3. 修正坐标(假如撞墙/撞到实体)
   4. 判断两个碰撞箱是否接触
   5. 判断当前物体是否着地(目前是简单版)
   6. 判断当前物体是否撞墙(目前是简单版)



## 一个角色 Player 类
1. 包含什么参数呢?
   1. 一个PhysicsData类型的参数myData，代表自身物理数据
   2. vector\<Animation> myAnimation 存储角色包含的动作序列
   3. 角色的血量
   4. 角色当前应当进行的动作序列
   5. 角色动作-优先级的映射
   
2. 初始化
   1. 初始化myData等物理数据
   2. 初始化myAnimation, 定义各个动作
   3. 初始化其他东西

3. 接受键盘的操作(压入动作序列)
   1. 跑步
   2. 跳跃
   3. 翻滚
   4. 下蹲
   5. 攻击
   6. 举盾

4. 角色自己的动作动画(未完待续，先做按键的)
   1. idle 静止
   2. idleRun 从静止到跑步
   3. runIdle (idleRun反向)
   4. run 普通跑步
   5. dash 冲刺
   6. endDashRun 从冲刺到奔跑
   7. jumpThrough 正常跳跃
   8. jumpTransition 正常跳跃跳跃结束后下落过程
   9. crouch 下蹲状态
   10. crouchIdle 蹲下到起立
   11. idleCrouch (crouchIdle反向)
   12. atkCodgA~C：砍刀与手枪攻击连招

5. 更新当前状态(暂时没有加入动画)
   1. 按照队首状态更新物理数据
   2. updatePosition()
   3. 队首状态更新动画系统(过一会写)

## 一套 Animation 系统
1. 一套集成的 AnimData 参数模板
   1. 一个哈希表存放动作类型-动作贴图集的映射
   2. 是否是某一资源的反转
   3. 是否是循环的动作
   4. 角色当前动作状态序列
   5. 角色目前正在做的动作(类型为T)
   6. 角色目前正在做的动作对应的sprite
   7. 当前动作播放到第几张

2. 一套动画的更新规则 AnimSystem
   1. 一定要定义一个计时器
   2. 根据当前动作序列切换播放素材
   3. 计时器达到某一时间就播放下一张
